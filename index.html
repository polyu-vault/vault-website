Vault
The Vault project aims to achieve a lower total cost of ownership by consolidating thousands of MPPDB tenants on to a shared hardware infrastructure, with a performance SLA that guarantees the tenants can obtain the query results as if they are executing their queries on dedicated machines.

MPPDB-as-a-Service:

Recently, massively parallel processing relational database systems (MPPDBs) like Vertica, Microsoft PDW, and Greenplum, have gained much momentum in the big data analytic market. With the advent of hosted cloud computing, we envision that the offering of MPPDB-as-a-Service (MPPDBaaS) will become attractive for companies having analytical tasks on only hundreds gigabytes to some ten terabytes of data because they can enjoy high-end parallel analytics at a cheap cost. This project is to develop Vault, a prototype implementation of MPPDB-as-a-service.



Vault Architecture:

Creating JDBC Application:

Vault provides JDBC Driver to open a database connection, execute SQL query and return the results from the JDBC Application.

According to the connection string provided in the JDBC application, Vault would route the request to the available MPPDB(s) for processing the query request.

Example 1: Use this template example when you need to create your own JDBC application interacting with Vault. 


          

import java.sql.*;
            
public class Example() {
  //JDBC driver name and database URL
  static final String JDBC_DRIVER = "vault.jdbc.driver.VaultDriver";
  static final String connectionString = "jdbc:vault://localhost:8080/db_srvr/1";
  				
  //Database credentials
  static final String USER = "username";
  static final String PASSWORD = "password";
			
  public static void main(String[] args){
  Connection conn = null;
  Statement stmt = null;
  try{
    //Register JDBC driver.
   	Class.forName(JDBC_DRIVER);
   	 					
   	//Open a connection
    System.out.println("Connecting to database...");
    conn = DriverManager.getConnection(connectionString, USER, PASSWORD);
      					
    //Execute a query
    System.out.println("Creating statement...");
    stmt = conn.createStatement();
    stmt.executeUpdate("CREATE TABLE Market(Date Date, Name varchar(64), PTID integer)");
    stmt.execute("COPY Market From LOCAL '/Users/Vault/sample.txt' DELIMITER '\t'");
    ResultSet rs = stmt.executeQuery("Select * FROM Market");
    stmt.executeUpdate("DROP TABLE Market");					
    } catch(SQLException se){
      //Handle errors for JDBC
      se.printStackTrace();
   	}catch(Exception e){
      //Handle errors for Class.forName
      e.printStackTrace();
   	}finally{
      //finally block used to close resources
      try{
        if(stmt!=null)
          stmt.close();
      	}catch(SQLException se2){
      	}// nothing we can do
      	try{
         if(conn!=null)
           conn.close();
      	}catch(SQLException se){
           se.printStackTrace();
      }//end finally try
    }//end try
  }//end main
}//end Example

Features

Objectives:

  • Shared-Process Consolidation
    • The Shared-Process approach is to share a single database instance among the tenants but each tenant can own its private set of tables
    • This approach leads to a higher consolidation effectiveness
  • Performance SLA Guarantee
    • Ensure that a tenant finishes its queries without feeling any performance delay in a specific percentage of time period.
  • High Availability & Load Balancing
    • Multiple MPPDBs in each tenant group allows workload balancing in Vault.
    • Vault is able to route the query request to the another available MPPDB which has the lowest workload.
    • Avoid the situation that some MPPDBs handle lots of query requests, but some MPPDBs in the same cluster are idle.

Functions:

  • Query Routing

    • Database user send their queries through the JDBC application, and then queries will be routed to the available MPPDBs. Database user do not need to specify the destination of the query because Vault will choose the most suitable MPPDB for you.

  • Tenant Consolidation

    • Deployment Master in Vault will consolidate tenants into different groups, therefore each of the groups share the same resources securely.

  • Real-time Analytics

    • As the Vault administrator, you can view different analysed data through dashboard, for example, tenant activity, tenant status, tenant group status and so on.

  • Renting Resources

    • The Vault user can create MPPDBs and database users on the cloud provided by the cloud operator through dashboard in order to enjoy our MPPDB as a service
